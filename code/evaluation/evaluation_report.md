### Projected cost for the full test set

- The sample set (20 claims) costs ~**$0.4945** under the routed config.
- `dataset/claims.csv` has 44 claims → projected ~**$1.0878** (list-price estimate; OpenRouter may add a small routing fee).
- Pricing assumptions (list, per 1M tokens): Haiku 4.5 $1/$5, Sonnet 4.6 $3/$15, Opus 4.8 $5/$25; cache reads ≈0.1×, cache writes ≈1.25×.

### Rate limits, batching, caching, retries

- **Prompt caching:** the system prompt + full evidence-requirements table form a byte-identical prefix across every call, so it is cache-written once and read at ~0.1× thereafter (see Cache-read tok above).
- **Image cost control:** images are downscaled to a 1280px long edge before encoding, capping image tokens.
- **On-disk response cache:** every call is keyed by (model, prompts, schema, image hashes); re-runs reuse cached responses, so iterating is free.
- **Concurrency / retries:** up to 6 concurrent claims; the SDK retries 429/5xx with exponential backoff (max_retries=4).
- **TPM/RPM:** at 44 test claims × ≤3 calls each (~130 requests, well under default RPM) the run is request-bound, not token-bound; bounded concurrency keeps us inside tier limits. OpenRouter spreads load across providers and supports provider sticky routing for cache hits; for larger volumes, lowering concurrency or routing to a cheaper model tier via env is the lever.

## 5. Final strategy

Ship **cost-first (Sonnet on every claim)** for `output.csv`. Every config above is *single-route* — exactly one Stage-2 call per claim, no Sonnet-then-Opus double-pay (the prior tiered config paid both on most rows). On this sample the single-route frontier is Sonnet-all (cheapest, matches the routed accuracy at lower cost) vs Opus-all (+accuracy, still ~25% under the old double-pay cost).

The up-front router (`router.py`, shown as the **routed** column with its triggers armed) is currently *dominated*: the only rows Opus uniquely fixes are image-hard, not text-hard, so cheap Stage-1/history signals can't catch them — it pays more than Sonnet-all for the same accuracy. It stays in the codebase, armed by a flag/env, because the planned CLIP **image-vs-claim** signal (vectorstore, ADR-0004) is exactly the cheap pre-Stage-2 trigger that would let routing reach Opus accuracy near Sonnet cost. Switch the default to accuracy-first any time with `python code/main.py --mode forced --model anthropic/claude-opus-4.8`.

All non-visual fields are computed by the deterministic rule layer (validated against 100% of samples in `code/tests/test_rules.py`).
