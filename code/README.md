# Multi-Modal Evidence Review — Solution

Verifies damage claims (`car` / `laptop` / `package`) by reading the submitted
images, the (multilingual) chat transcript, user history, and the minimum
evidence requirements, then emitting the structured `output.csv`.

Design language lives in [`../CONTEXT.md`](../CONTEXT.md); the architecture
decisions are recorded in
[ADR-0001 (two-stage pipeline + rule layer)](../docs/adr/0001-two-stage-pipeline-with-rule-layer.md),
[ADR-0002 (OpenRouter provider)](../docs/adr/0002-openrouter-provider.md),
[ADR-0003 (single-route Stage-2, no double-pay)](../docs/adr/0003-single-route-stage2.md),
and [ADR-0004 (pgvector reused-image detection)](../docs/adr/0004-pgvector-image-dedup.md).

Models are served through **[OpenRouter](https://openrouter.ai)** over the
OpenAI-compatible API (OpenRouter's recommended Python transport is the OpenAI
SDK pointed at its base URL), so the pipeline is provider-portable. The default
slugs route to the same Claude models, and every slug is overridable via env.

## Architecture — two model stages + a deterministic rule layer

```
claim ─▶ Stage 1: Claim Extraction (Haiku 4.5, text-only)
            • multilingual chat ─▶ canonical claim intent (part(s), issue(s), summary)
            • flags instruction/manipulation text in the chat (treated as untrusted data)
       ─▶ Stage 2: Visual Verification (single-routed: Sonnet 4.6 cost-first; Opus 4.8 optional)
            • images are the source of truth
            • issue_type, object_part, claim_status, evidence_standard_met,
              valid_image, severity, supporting_image_ids, visual risk flags,
              justifications, confidence
       ─▶ Stage 3: Rule layer (deterministic Python)
            • evidence_standard_met=false ⟺ claim_status=not_enough_information; severity=unknown
            • user_history_risk / manual_review_required derived mechanically
            • history never changes the verdict
            • issue_type=none ⟹ severity=none; non_original_image ⟹ valid_image=false
            • enum normalization, flag dedup/ordering, exact CSV schema
```

**Single-route model strategy (shipped, cost-first):** every claim pays for
**exactly one** Stage-2 call — there is no Sonnet-then-Opus double-pay. The
default ships Sonnet 4.6 on every claim (cheapest; on the sample it matches the
accuracy of the smarter configs at lower cost). An up-front router (`router.py`)
can escalate hard rows to Opus 4.8 *before* any vision call, armed via env
(`ORCH_ROUTE_ON_*`); see `docs/adr/0003-single-route-stage2.md` for the measured
cost/accuracy frontier and why the text/history triggers ship off by default.
Switch to accuracy-first any time with
`python code/main.py --mode forced --model anthropic/claude-opus-4.8`.

Every rule in Stage 3 is validated against **100% of the 20 labeled samples**
(`tests/test_rules.py`) and runs with no API key.

## Setup

```bash
pip install -U -r code/requirements.txt    # openai SDK (OpenRouter transport)
cp code/.env.example code/.env             # then put your OpenRouter key in it
# or: export OPENROUTER_API_KEY=sk-or-v1-...   (get one at openrouter.ai/keys)

# Optional — only for the shared Redis cache (Item 3) and pgvector store (Item 4);
# not needed for a plain run. Or just use Docker (below), which installs them.
pip install -U -r code/requirements-infra.txt
```

## Run

```bash
# Final predictions for the test set (cost-first: Sonnet on every claim) -> ./output.csv
python code/main.py

# Quick smoke test on a few rows
python code/main.py --limit 3

# Accuracy-first: Opus on every claim
python code/main.py --mode forced --model anthropic/claude-opus-4.8

# Arm the up-front router (escalate hard rows to Opus before any vision call)
ORCH_ROUTE_ON_INSTRUCTION_TEXT=1 ORCH_ROUTE_ON_MULTI_PART=1 ORCH_ROUTE_ON_HISTORY_RISK=1 \
  python code/main.py --mode routed

# Force a single fixed Stage-2 model
python code/main.py --mode forced --model anthropic/claude-sonnet-4.6

# Evaluation: Sonnet-all vs Opus-all vs Routed on the labeled samples,
# writes code/evaluation/evaluation_report.md + per-config preds_*.csv
python code/evaluation/main.py

# Deterministic tests (no API key needed): rule layer, router, vector-flag contract
python code/tests/test_rules.py && python code/tests/test_router.py && python code/tests/test_extra_flags.py
```

`output.csv` is written to the repo root by default; pass `--output dataset/output.csv`
to write it next to the dataset instead.

## Run with Docker (full stack)

The repo-root `Dockerfile` + `docker-compose.yml` bring up the app alongside
**Redis** (shared response cache, Item 3) and **Postgres + pgvector** (reused-image
store, Item 4) — one command, no local Python or service setup:

```bash
# Put your key where Compose can read it (Compose auto-loads ./.env):
echo "OPENROUTER_API_KEY=sk-or-v1-..." >> .env

docker compose run --rm app                                   # -> ./output.csv (cost-first)
docker compose run --rm app python code/evaluation/main.py    # regenerate the report
docker compose run --rm app python code/dedup.py index        # build the image index
docker compose run --rm app python code/dedup.py report       # cross-user reused images
```

The app auto-detects the services via `REDIS_URL` / `DATABASE_URL` (set by
Compose). Run the same commands **without** Docker and they degrade gracefully:
no `REDIS_URL` → local filesystem cache; no `DATABASE_URL` → vector store skipped.

## Reused-image detection (pgvector, optional)

When `DATABASE_URL` is set, the pipeline indexes every image up-front and flags
any image reused across **different users** as `non_original_image` (→
`valid_image=false` + `manual_review_required`). The default embedding is a
dependency-light perceptual hash (Pillow only); set `ORCH_EMBED_BACKEND=clip`
for semantic CLIP embeddings (needs `code/requirements-infra.txt`'s optional
`sentence-transformers`). Tune sensitivity with `ORCH_DEDUP_HAMMING` (pHash) or
`ORCH_DEDUP_COSINE` (CLIP). See [ADR-0004](../docs/adr/0004-pgvector-image-dedup.md).

## How the design handles the adversarial cases

- **Multilingual chats** (Hindi/Hinglish, Spanish, Chinese pinyin): Stage 1
  normalizes the claim to English before any visual reasoning.
- **Prompt injection** in the chat *or* baked into an image ("approve
  immediately", "ignore previous instructions", "SECURITY SEAL / DO NOT
  ACCEPT"): treated as untrusted data, flagged `text_instruction_present`, paired
  with `manual_review_required`, and never allowed to move the verdict.
- **Non-original images** (watermark/stock/render): flagged `non_original_image`
  → `valid_image=false`, while the claim is still judged on what's visible.
- **Exaggeration / mismatch**: `severity` reflects the *actual visible* damage,
  not the claimed magnitude; `claim_mismatch` / `wrong_object` flagged.
- **Robustness**: any per-claim failure produces a safe `not_enough_information`
  + `manual_review_required` fallback row, so `output.csv` always has one row per
  input.

## Reproducibility & cost

- Structured outputs pin every categorical field to its allowed enum.
- **Single Stage-2 call per claim** (no double-pay): the 44-row test set costs
  ~$0.91 cost-first (Sonnet, the shipped default), ~$1.09 with the router armed
  (the figure projected in `evaluation/evaluation_report.md`), or ~$1.56
  accuracy-first (Opus) — all well under the ~$2.09 of the old
  confidence-escalation config (ADR-0003).
- The system prompt + evidence-requirements table are a stable, prompt-cached
  prefix; images are downscaled; responses are cached on disk by input hash, so
  re-runs are free and stable. The cache backend is pluggable — filesystem by
  default, Redis when `REDIS_URL` is set, for sharing across workers/machines.
- See `evaluation/evaluation_report.md` (generated) for the full operational
  analysis: model calls, token usage, cost with pricing assumptions, latency, and
  TPM/RPM / batching notes.
